# Helios TCE Adapter — Package Manifest Schema

## Purpose

The package manifest (`package-manifest.json`) describes the contents, provenance, and verification data for a TCE Helios adapter package. It is generated by `New-HeliosAdapterPackage.ps1` and verified by `Test-HeliosAdapterPackage.ps1`.

## Schema

  "package_name": "helios-tce-adapter",
  "package_version": "0.3.98",

  "source_branch": "helios-integrity-adapter",
  "source_commit": "783193e...",
  "build_timestamp_utc": "2026-06-27T14:30:00Z",
  "files": [
    {
      "path": "HeliosIntegrityBridge.ps1",

## Field Definitions

### Package Identity

| Field | Type | Description |
|---|---|---|
| `schema_version` | string | Always `"helios-adapter-package.v1"` |
| `package_name` | string | Package identifier: `"helios-tce-adapter"` |
| `package_version` | string | Semantic version (e.g., `"0.3.98"`) |

### Provenance

| Field | Type | Description |
|---|---|---|
| `source_repo` | string | Repository name |
| `source_branch` | string | Branch the package was built from |
| `source_commit` | string | Full commit hash at build time |
| `build_timestamp_utc` | string | ISO 8601 UTC timestamp of package creation |

### Contents

| Field | Type | Description |
|---|---|---|
| `files` | array | List of files in the package with path, role, and required flag |
| `file_hashes` | object | SHA-256 hash for each file, keyed by relative path |

#### File Roles

| Role | Description |
|---|---|
| `bridge_source` | Source-of-truth bridge implementation |
| `schema` | JSON Schema definition |
| `tool` | PowerShell tool script |
| `doc` | Documentation |
| `test` | Test suite |
| `evidence` | Gap-test evidence artifact |
| `readme` | Package or adapter README |
| `manifest` | This package manifest |
| `checksums` | Checksums file |

### Compatibility

| Field | Type | Description |
|---|---|---|
| `required_helios_version` | string | Minimum Helios runtime version |
| `required_tce_adapter_version` | string | Adapter version this package provides |
| `supported_platforms` | array | `["windows", "linux", "macos"]` |

### Installer Integration

| Field | Type | Description |
|---|---|---|
| `installer_entrypoints` | object | Relative paths to installer tools |

### Smoke Tests

| Field | Type | Description |
|---|---|---|
| `smoke_tests` | array | Tests to run after installation |
| `smoke_tests[].name` | string | Test identifier |
| `smoke_tests[].description` | string | What the test verifies |
| `smoke_tests[].expected_verdict` | string | Expected Helios verdict |
| `smoke_tests[].expected_evidence` | array | Evidence files expected (allow tests only) |

### Trust Boundaries

| Field | Type | Description |
|---|---|---|
| `trust_boundaries` | object | Documents trust ownership for each boundary |
| `trust_boundaries.bridge_source_of_truth` | string | Where the canonical bridge lives |
| `trust_boundaries.bridge_vendor_copy` | string | Where Helios consumes the bridge |
| `trust_boundaries.manifest_authority` | string | Who generates the runtime manifest |
| `trust_boundaries.settings_activation` | string | What approval is needed for hook activation |
| `trust_boundaries.package_integrity` | string | How package integrity is verified |

## Checksums File

The `checksums.sha256` file is a plain-text file listing SHA-256 hashes for all package files:

```
<hash>  HeliosIntegrityBridge.ps1
<hash>  README.md
<hash>  docs/tce-helios-integrity-adapter-spec.md
...
```

Format: `<lowercase_hex_sha256>  <relative_path>` (two-space separator, matching `sha256sum` output format).

Generated by `New-HeliosAdapterPackage.ps1`. Verified by `Test-HeliosAdapterPackage.ps1`.
